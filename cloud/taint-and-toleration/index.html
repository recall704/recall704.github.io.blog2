k8s 中的污染 Taint 与容忍 Toleration

Posted on January 18, 2018

一、 taint 污染

taint 其实和 k8s 中的 label 概念很相似，它有一个类似 label 一样的 key-value 键值对以及对应的属性。

kubectl taint nodes node1 aaa=bbb:NoSchedule

其中： aaa 代表 taint 的 key
bbb 代表 taint 的 value
NoSchedule 代码这个 taint 对应的影响，它可以有三个值

值 含义
NoSchedule 不允许新的 pod 调度到该 node 上，除非 toleration 条件满足
PreferNoSchedule 尽可能不要调度到该 node 上，除非 toleration 条件满足
NoExecute 驱逐该节点上的 pod 到其它节点，除了 toleration 条件满足的 pod

二、toleration 容忍

增加了 taint 的 node，不再允许调度 pod 到上面运行，但是有时候我们又希望自己的 pod 能运行在上面，那这个时候，只要我们的 pod 增加对应的容忍的参数，与 taint 的 key-value 对应，就行了。

tolerations:
- key: "aaa"
  operator: "Equal"
  value: "bbb"
  effect: "NoSchedule"
operator 值 含义
Equal 只有 key-value 都和 taint 一致，才算满足容忍条件，pod 才能跑在上面
Exists 只要 key 存在，就算满足容忍条件，value 可以为空

三、常用使用案例

  1. 为了保证 master 节点的稳定性，我们可以对 master 节点添加 taint，除了日志和监控需要的 pod 添加了对应的 toleration 之外，不允许别的 pod 运行在 master 节点上
  2. 节点升级维护，当节点需要维护时，可以给节点增加 taint 且 effect 的值为 NoExecute，pod 调度到其它节点之后再开始升级维护。

Q：taint 会影响到 daemonset 吗？
A：会，taint 是针对所有 pod 的，daemonset 最终也会生成对应的 pod，所以 daemonset 配置的时候需要设置对应的 toleration。

Q: 如何删除 taint
A: kubectl taint nodes node1 key:NoSchedule-
