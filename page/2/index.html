k8s 中的污染 Taint 与容忍 Toleration

@recall704 · Mar 11, 2018 · 2 min read

之前的内部分享中我提到过，pod 最终会被调度到某个具体的 node 上运行。那么有没有某种方式，限制 pod，不让它在某些 node 上运行呢？ 或者说，只允许某些 pod 在某些 node 上运行呢？ taint 和 toleration 了解一下。 一、 taint （污染） 首先一点，taint 是针对 node 而言的，给 node 增加 taint 的命令如下 kubectl taint nodes node1 aaa=bbb:NoSchedule 其中： 1. aaa 代表 taint 的 key 2. bbb 代表 taint 的 value 3. NoSchedule 代码这个 taint 对应的影响，它可以有三个值 值 含义 NoSchedule 不允许新的 pod 调度到该 node 上，除非 toleration 条件满足；不影响已经运行在该 node 上的 pod PreferNoSchedule 尽可能不要调度到该 node 上，除非 toleration 条件满足；不影响已经运行在该 node 上的 pod NoExecute 驱逐该节点上的 pod 到其它节点，除了 toleration 条件满足的 pod；影响已经运行在该 node 上的所有 pod 我们了解到一点，那就是增加了 taint 的 node，pod 就不会调度到它上面运行了。
